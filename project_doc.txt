CertificateTrustSystem
    Класс реализующий иерархию доверия.
    Методы класса:
        __init__(self, root_cert_path, revocation_list_path=None)
            Загружает в память корневой сертификат и CRL и инициализирует иерархию доверия.

        add_certificate_authority(self, cert_path)
            Добавляет промежуточные сертификаты УЦ (УЦ - удостоверяющий центр) в цепочку доверия
            Проверяет цепочку сертификатов перед добавлением (см _validate_certificate_chain)

        verify_signed_application(self, app_path, signature_path, cert_path)
            Осуществляет проверку приложения с цифровой подписью (Проверяет доступность файлов, 
            Валидирует цепочку сертификатов, Проверяет цифровую подпись, Проверяет целостность приложения через хеширование)

        _validate_certificate_chain(self, certificate)
            Проверяет цепочку сертификатов до корневого

        _validate_certificate_period(self, certificate)
            Проверяет, не истёк ли срок действия сертификата, и находится ли он в CRL(Certificate Revocation List, список отозванных сертификатов)

        _validate_digital_signature(self, data_path, signature_path, certificate)   
            Проверяет цифровые подписи с использованием открытого ключа сертификата (Пробует несколько схем подписи)
        
        _check_application_integrity(self, file_path)
            Генерирует SHA-256 хеш данных приложения
            Используется для проверки, что приложение не было изменено

        _load_x509_certificate, _load_crl, _verify_file_accessibility - методы для проверки доступности и загрузки сертификатов

TrustVerificationError
    Пользовательский класс исключения для ошибок проверки доверия

MockTrustSystem
    Класс для тестирования зависимостей.

TrustSystemTests
    Класс для запуска модульных тестов с использованием mock компонентов
    Тестирует сценарии, включая:
        Инициализацию цепочки доверия
        Добавление промежуточного УЦ
        Отклонение просроченных сертификатов
        Обнаружение отозванных сертификатов
        Обработку недействительных подписей
        Отклонение недоверенных сертификатов
        Обнаружение повреждённых приложений
    Для каждого сценария введён соответствующий метод

IntegrationTests
    Класс для запуска тестов с реальными файлами сертификатов
    Осуществляет сквозную проверку приложений, валидацию цепочки сертификатов, проверку цифровой подписи.
    Класс выполняет тесты и выводит отчёт об ошибках при сбоях.
